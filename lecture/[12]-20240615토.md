# 15장 쓰레드
## 15.1 프로그램, 프로세스, 쓰레드
### 프로그램
- 하드디스크에 저장된 파일들의 모임
### 프로세스
- 메모리상에 로딩된 프로그램을 의미
### 메모리
- 그때그때 필요한 부분만 동적으로 로딩
### 멀티 프로세스 
- 동일한 프로그램을 메모리에 2번 로딩하면 2개의 프로세스가 동작
    - ex) 워드 파일 2개 열면, 워드 프로그램이 메모리에 2번 로딩 돼 멀티 프로세스로 동작
### 쓰레드의 개념 
- 프로세스만 CPU를 사용할 수 있다는 말. 
    - 그러나 좀 더 들여다보면, 실제 CPU를 사용하는 것은 프로세스 내부의 쓰레드라는 것을 알 수 있다.
    - 프로세스가 쓰레드를 갖고 있으므로, 외부에서 보면 프로세스가 CPU를 사용하는 것처럼 보였던 것 뿐이다. 
> CPU를 사용하는 최소 단위

### 자바 프로그램에서의 쓰레드
- 자바로 작성한 프로그램을 실행하면, 메모리로 로딩돼 프로세스 상태가 된다. 
- .class 파일을 실행하면, JVM은 main 쓰레드를 생성한다.
    - main()에서 작성한 내용이, main 쓰레드에서 동작한다.
    - 만일 main 쓰레드의 내부에서 다음과 같이 2개의 쓰레드를 생성해 실행하면, 동시에 2개 이상의 쓰레드가 동작하게 되는데, `멀티 쓰레드` 프로세스라고 한다.  
### 멀티 쓰레드의 필요성
- 쓰레드는 CPU를 사용하는 최소 단위
    - 멀티 쓰레드는 2개 이상의 쓰레드가 동시에 CPU를 사용한다느 소리 
### 쓰레드는 정말 동시에 수행될까?
- ex) 음악을 들으면서 웹 서핑을 하는 것 
- 내 컴퓨터에는 단 하나의 CPU만 있는데, 어떻게 각 쓰레드는 동시에 실행되는 걸까?
    - 동시성
    - 병렬성
- 단일 쓰레드: 2개 작업을 처리할 경우엔 작업을 순차적으로 처리된다. 
### 동시성
- 처리할 작업의 수가 CPU의 코어 수보다 많을 때이다. 
    - CPU의 코어는 1개인데, 동시에 처리해야 할 작업이 2개일 때 
- 작업 쓰레드의 요청 작업을 번갈아가면서 실행한다. 
- 매우 짧은 간격으로 교차 실행하기 때문에, 사용자는 두 작업이 마치 동시에 실행되는 것처럼 보이는 것이다.
    - 엄밀히 말하면, 두 작업이 동시에 실행되는 것이 아니라, 동시에 실행되는 것처럼 보이도록 하는 방식이다. 
### 병렬성
    - CPU의 코어 수가 작업 수보다 많을 때 -> 각각의 코어에 할당해 동시에 실행할 수 있다.
    - ex) 작업이 3개, CPU의 코어 수 4개 -> 쓰레드 병렬성이 적용됨
- ex) 작업 수 6개, 코어 2개 -> 쓰레드의 동시성과 병렬성이 함께 적용 
    - 작업이 2개의 코어에 나뉘어 할당(병렬성), 각각의 코어는 할당된 작업을 번갈아 실행(동시성)
## 15.2 쓰레드의 생성 및 실행
1. Thread 클래스를 상속 받아 run() 메서드를 오버라이딩(재정의)
- 메서드 내부에서 작성된 내용이 바로 CPU를 독립적으로 사용하면서 동시에 실행되는 것이다. 
- Runnable 인터페이스를 구현한 Runnable 객체를 생성한다. 
- 인터페이스는 추상 메서드로 run()메서드를 갖고 있다.
- 인터페이스 객체 생성 과정 중 run() 메서드를 구현해야 한다. 
2. Thread 객체를 생성할 때, 앞 단계에서 생성한 Runnable 객체를 생성자로 전달하는 것이다. 즉, 2가지 생성 방법 모두 run() 메서드를 재정의하고 있고, 결과적으로 Thread 객체를 생성한다. 
> Thread 객체 내의 start() 메서드를 호출하는 것이다. 
> - run()의 내용을 실행하기 위해서는 반드시 start() 메서드를 호출해야 한다는 것.
