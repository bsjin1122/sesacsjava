## 3장

- 서버에서 자동으로 생성하는 쿠키
    - 정해진 쿠키가 없는 경우 자동으로 발행
    - WAS에서 발행되며, 이름은 WAS마다 고유한 이름을 사용해 쿠키를 생성(대부분 JSESSIONID)

- HTTP는 기본적으로 무상태 -> 과거의 요청 기록 알 수 없음 
- 세션 트랙킹: 과거의 방문 기록을 추적하는 기법 

- 요청(Request)과 응답(Response)을 주고받는다. 
- 쿠키: 이름(name)과 값(value) , 개발자도구의 애플리케이션 메뉴 

### 쿠키 주고받는 시나리오
- 브라우저에서 최초 서버 호출 시, 해당 서버에서 발행한 쿠키가 없다면 브라우저에서는 아무것도 전송하지 않는다.
- 서버에서는 응답 메시지 보낼 떄, 브라우저에게 쿠키를 보내주는데, Set-Cookie라는 http 헤더를 이용한다.
- 브라우저는 쿠키를 받은 후에 이에 대한 정보를 읽고, 이를 파일 형태로 보관/메모리상에서만 처리할 것인지를 결정한다.
    - 이 판단은 쿠키에 있는 "유효기간"을 보고 판단한다.
- 브라우저가 보관하는 쿠키는 다음에 다시 브라우저가 서버에 요청(Request)할 때, http 헤더에 Cookie 라는 헤더 이름과 함께 전달한다.
- 서버에서는 필요에 따라 브라우저가 보낸 쿠키를 읽고, 이를 사용한다. 

### 쿠키 생성하는 방법
- 서버에서 쿠키를 발행하는 것은 버서에서 자동으로 발행되는 방식 / 개발자가 코드를 통해 직접 발행하는 2가지 방식
- 서버에서 자동으로 생성하는 쿠키
    - 응답 메시지를 작성할 때 정해진 쿠키가 없는 경우, 자동으로 발행 - WAS에서 발행, 이름은 WAS마다
